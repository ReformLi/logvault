## 环境变量说明

以下所有环境变量需要在 Vercel 项目控制台的 **Settings → Environment Variables** 中配置。本地开发时写入 `.env.local`。

### `NEXTAUTH_SECRET`（必需）

NextAuth 会话加密密钥，任意长度的随机字符串。

```bash
# Linux/macOS 生成
openssl rand -base64 32
```

示例：`coqms5DnpkZ2z6oYgKqHk4DBN7dHOvAWxOiDQDjI9zY=`

---

### `AUTH_GITHUB_ID` / `AUTH_GITHUB_SECRET`（必需）

GitHub OAuth App 的凭据。也支持旧名称 `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET`。

**获取方式：**

1. 访问 https://github.com/settings/developers → **OAuth Apps** → **New OAuth App**
2. 填写：
   - **Application name**：`LogVault`（可自定义）
   - **Homepage URL**：部署后的域名（本地开发填 `http://localhost:3000`）
   - **Authorization callback URL**：`<部署域名>/api/auth/callback/github`
3. 创建后复制 **Client ID** 和 **Client Secret**

---

### `ENCRYPTION_SECRET`（必需）

AES-256-GCM 加密密钥，64 位十六进制字符串（32 字节）。

```bash
openssl rand -hex 32
```

示例：`92e28f57325bfdc73e29efba4481ce24b884735ef8babf4f0603c5f8abb391c5`

⚠️ **此密钥丢失后已加密的日志将无法恢复，务必妥善保管。**

---

### `VERCEL_TOKEN`（必需）

Vercel API 访问令牌，用于调用部署查询和日志拉取接口。

**获取方式：**

1. 登录 https://vercel.com/account/settings/tokens
2. 点击 **Create** → 填写名称（如 `logvault`）
3. Scope 选择目标团队或个人账号
4. 复制生成的 token（格式：`vcp_xxxxxxxxxxxx`）

---

### `VERCEL_PROJECT_ID`（必需）

目标 Vercel 项目的 ID。

**获取方式：**

1. 打开目标项目的 Vercel 控制台
2. 进入 **Settings → General**
3. 找到 **Project ID** 并复制
4. 格式示例：`prj_FpXLI9nX6g411yGiJhpBZ8SG5wxC`

> 或者使用 `VERCEL_PROJECT_NAME` 替代，传入项目名称（如 `my-app`）。至少需要配置其中一个。

---

### `VERCEL_OWNER_ID`（必需 — GitHub Actions 需要手动配置）

Vercel 团队或个人用户 ID，用于日志 API 调用。

应用代码中会自动检测（先查询所属 Team，再回退到个人用户 ID），但在 GitHub Actions 工作流中需要**手动配置到 Secrets**。

**获取方式：**

```bash
# 先获取团队列表
curl -H "Authorization: Bearer <VERCEL_TOKEN>" https://api.vercel.com/v2/teams

# 或获取个人用户信息
curl -H "Authorization: Bearer <VERCEL_TOKEN>" https://api.vercel.com/v2/user
```

示例：`team_7JzlPPvVT9czhQwFar8oEXAN`

---

### `POSTGRES_URL`（Vercel Postgres 自动注入）

PostgreSQL 数据库连接字符串。

- **Vercel 部署**：在 Vercel 控制台中创建 **Storage → Postgres** 数据库，连接字符串会自动注入到环境变量 `POSTGRES_URL`（无需手动配置）
- **本地开发**：使用本地 PostgreSQL 实例，格式：
  
  ```
  postgresql://用户名:密码@127.0.0.1:5432/数据库名
  ```

---

### `USE_LOCAL_DB`（本地开发用）

- `true` — 使用 `POSTGRES_URL` 直连本地 PostgreSQL，同时 blob 写入 `public/uploads/`
- `false` 或未设置 — 使用 `@vercel/postgres`（默认，云端部署用）

---

### `ALLOWED_EMAILS`（必需）

允许登录的 GitHub 邮箱白名单，逗号分隔。

示例：`alice@gmail.com,bob@outlook.com`

⚠️ **不配置时所有登录都被拒绝**，必须至少配置一个邮箱才能正常登录。

---

### `ADMIN_EMAILS`（可选）

管理员邮箱白名单，逗号分隔。拥有系统设置、删除日志、审计日志页面权限。

不配置时所有登录用户都不是管理员。

---

### `CRON_SECRET`（强烈建议）

保护 `/api/fetch/cron`、`/api/cleanup` 和 `/api/logs/store` 接口的 Bearer Token。

```bash
openssl rand -hex 32
```

⚠️ **不配置时这三个接口会直接返回 500 错误**，定时任务和清理功能将不可用。

## 数据库初始化

### 方式一：Vercel Postgres（推荐）

1. 在 Vercel 控制台进入项目 → **Storage** → **Connect Store** → **Create New Database** → **Postgres**
2. 创建完成后，数据库连接变量自动注入
3. 打开 **Storage → Postgres → Data** 页面
4. 复制 `lib/schema.sql` 中的内容，在 SQL 编辑器中执行

### 方式二：本地 PostgreSQL

```bash
# 创建数据库
createdb logvault_local

# 执行建表脚本
psql -U postgres -d logvault_local -f lib/schema.sql
```

## Vercel 部署步骤

```bash
# 1. 安装 Vercel CLI（如未安装）
npm i -g vercel

# 2. 登录 Vercel
vercel login

# 3. 在项目目录中关联 Vercel 项目
vercel link

# 4. 在 Vercel 控制台配置所有环境变量
#    Settings → Environment Variables → 逐一添加

# 5. 部署
vercel --prod
```

部署后首次访问会自动跳转到 GitHub OAuth 登录。确认使用的邮箱在 `ALLOWED_EMAILS` 中即可。

### 配置定时任务（GitHub Actions + Vercel Cron）

Vercel Hobby 计划的 Cron Jobs 每天只能触发一次，且 **从 Vercel 内部调用 Vercel API 会返回受限数据**（无函数日志内容）。因此：

- **日志抓取**：改用 GitHub Actions，每 30 分钟从**外部**直连 Vercel API，获取完整数据后调用 APP 的 `/api/logs/store` 存储
- **数据清理**：保留在 Vercel Cron（`vercel.json`），每天凌晨执行一次

#### 步骤一：配置 GitHub Secrets

在 **GitHub 仓库 Settings → Secrets and variables → Actions** 中添加：

| Secret              | 说明                                     |
| ------------------- | -------------------------------------- |
| `VERCEL_TOKEN`      | Vercel API 访问令牌                        |
| `VERCEL_PROJECT_ID` | 目标 Vercel 项目 ID                        |
| `VERCEL_OWNER_ID`   | Vercel 团队/用户 ID                        |
| `CRON_SECRET`       | 与 Vercel 环境变量中的 `CRON_SECRET` 一致       |
| `APP_URL`           | 部署后的域名，如 `https://logvault.vercel.app` |

#### 步骤二：推送工作流文件

项目中已包含 `.github/workflows/fetch-logs.yml`，内容如下：

```yaml
name: Fetch Vercel Logs

on:
  schedule:
    - cron: '*/30 * * * *'
  workflow_dispatch:

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - name: Get latest deployment
        run: |
          NOW=$(date +%s%3N)
          YESTERDAY=$((NOW - 86400000))
          echo "START_DATE=$YESTERDAY" >> $GITHUB_ENV
          echo "END_DATE=$NOW" >> $GITHUB_ENV

          DEPLOYMENT=$(curl -s --fail \
            "https://api.vercel.com/v6/deployments?limit=1&state=READY&target=production&projectId=${{ secrets.VERCEL_PROJECT_ID }}" \
            -H "Authorization: Bearer ${{ secrets.VERCEL_TOKEN }}")
          DEPLOYMENT_ID=$(echo "$DEPLOYMENT" | jq -r '.deployments[0].uid // empty')
          echo "DEPLOYMENT_ID=$DEPLOYMENT_ID" >> $GITHUB_ENV

      - name: Fetch request logs from Vercel API
        if: env.DEPLOYMENT_ID != ''
        run: |
          curl -s --fail \
            "https://vercel.com/api/logs/request-logs?projectId=${{ secrets.VERCEL_PROJECT_ID }}&ownerId=${{ secrets.VERCEL_OWNER_ID }}&startDate=${{ env.START_DATE }}&endDate=${{ env.END_DATE }}&deploymentId=${{ env.DEPLOYMENT_ID }}&environment=production&limit=200" \
            -H "Authorization: Bearer ${{ secrets.VERCEL_TOKEN }}" \
            -o raw-logs.json
          echo "Rows: $(jq '.rows | length' raw-logs.json)"

      - name: Store logs via app
        if: env.DEPLOYMENT_ID != ''
        run: |
          curl -s --fail -X POST "${{ secrets.APP_URL }}/api/logs/store" \
            -H "Authorization: Bearer ${{ secrets.CRON_SECRET }}" \
            -H "Content-Type: application/json" \
            -d "$(jq -c --arg id "${{ env.DEPLOYMENT_ID }}" '{deploymentId: $id, rows: .rows}' raw-logs.json)"
```

#### 步骤三：确认 Vercel Cron 配置

`vercel.json` 中已配置清理任务，每天凌晨执行：

```json
{
  "crons": [
    {
      "path": "/api/cleanup",
      "schedule": "0 0 * * *"
    }
  ]
}
```

推送代码后 Vercel 会自动部署，无需手动配置 cron。

## 本地开发

```bash
# 1. 克隆仓库
git clone <仓库地址> && cd logvault

# 2. 安装依赖
npm install

# 3. 复制环境变量模板
cp .env.local.example .env.local
# 编辑 .env.local 填入真实值

# 4. 确保 PostgreSQL 运行中并已初始化数据库
psql -U postgres -d logvault_local -f lib/schema.sql

# 5. 启动开发服务器
npm run dev
```

访问 `http://localhost:3000` 即可。
